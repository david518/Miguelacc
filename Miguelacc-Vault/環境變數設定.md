# 🔐 環境變數設定

回到 [[00-總覽 (MOC)]]。

所有變數放在專案根目錄的 `.env` 檔。

> [!danger] 以下為「本機測試用」假值
> `JWT_SECRET` 與 admin 帳密是開發測試用的隨意值，**正式上線前務必更換**，且 `.env` 不應提交到公開的版本庫。

## 完整 `.env` 內容

```ini
# 資料庫
DATABASE_URL="file:./prisma/dev.db"

# 網站基底網址（金流回呼用）
NEXT_PUBLIC_BASE_URL="http://localhost:3000"

# 後台登入（用於 /admin/login）
JWT_SECRET="local-dev-secret-change-me-please-32chars"
ADMIN_EMAIL="admin@example.com"
ADMIN_PASSWORD="admin1234"

# 綠界 ECPay 金流（測試環境，官方公開測試帳號）
ECPAY_IS_STAGE="true"
ECPAY_MERCHANT_ID="3002607"
ECPAY_HASH_KEY="pwFHCqoQZGmho4w6"
ECPAY_HASH_IV="EkRm7iFT261dpevs"

# 綠界 ECPay 物流（C2C 超商取貨，官方測試帳號，與金流不同）
ECPAY_LOGISTICS_MERCHANT_ID="2000933"
ECPAY_LOGISTICS_HASH_KEY="XBERn1YOvpM9nfZc"
ECPAY_LOGISTICS_HASH_IV="h1ONHk4P4yqbl5LK"
```

> [!note] 物流與金流是不同帳號
> `ECPAY_IS_STAGE` 同時控制金流與物流的測試/正式切換。憑證細節見 [[綠界測試金流]] 與 [[超商取貨與商品上架]]。

## 變數說明

| 變數 | 用途 | 程式位置 |
|------|------|----------|
| `DATABASE_URL` | SQLite 資料庫路徑 | `src/lib/prisma.ts` |
| `NEXT_PUBLIC_BASE_URL` | 結帳後綠界回呼網址 | `src/app/checkout/actions.ts` |
| `JWT_SECRET` | 簽發 admin session token（**未設定會拋錯**） | `src/lib/auth.ts` |
| `ADMIN_EMAIL` | 後台登入帳號 | `src/lib/auth.ts` |
| `ADMIN_PASSWORD` | 後台登入密碼 | `src/lib/auth.ts` |
| `ECPAY_IS_STAGE` | `true`=測試環境 / `false`=正式 | `src/lib/ecpay.ts` |
| `ECPAY_MERCHANT_ID` | 綠界商店代號 | `src/lib/ecpay.ts` |
| `ECPAY_HASH_KEY` | 綠界 HashKey | `src/lib/ecpay.ts` |
| `ECPAY_HASH_IV` | 綠界金流 HashIV | `src/lib/ecpay.ts` |
| `ECPAY_LOGISTICS_MERCHANT_ID` | 綠界物流商店代號 | `src/lib/ecpay-logistics.ts` |
| `ECPAY_LOGISTICS_HASH_KEY` | 綠界物流 HashKey | `src/lib/ecpay-logistics.ts` |
| `ECPAY_LOGISTICS_HASH_IV` | 綠界物流 HashIV | `src/lib/ecpay-logistics.ts` |

相關：[[綠界測試金流]]、[[後台管理]]。

> [!important] 改完要重啟
> `.env` 修改後一定要重啟 `npm run dev` 才會載入新值。
