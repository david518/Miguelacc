# 🛠️ 本機部署指南

回到 [[00-總覽 (MOC)]]。

## 前置需求

- **Node.js** v22+（實測 v22.15.0）
- **npm** v10+（實測 10.9.2）
- 專案路徑：`C:\Miguelacc`

## 首次部署步驟

```bash
cd C:\Miguelacc

# 1. 安裝依賴（從 lockfile 完整安裝）
npm ci

# 2. 生成 Prisma Client（輸出到 src/generated/prisma）
npx prisma generate

# 3. 確認資料庫 schema 為最新
npx prisma migrate status

# 4. 啟動開發伺服器
npm run dev
```

啟動成功後會看到：

```
▲ Next.js 16.2.2 (Turbopack)
- Local:   http://localhost:3000
✓ Ready in ~700ms
```

> [!warning] 開始前先設定 `.env`
> 缺少 `JWT_SECRET` 等變數會讓 admin 登入直接拋錯。請先完成 [[環境變數設定]]。

## 常用指令

| 指令 | 說明 |
|------|------|
| `npm run dev` | 啟動開發伺服器（含熱更新 Fast Refresh） |
| `npm run build` | 編譯正式版 |
| `npm run start` | 啟動正式版（需先 build） |
| `npm run lint` | ESLint 檢查 |
| `npx prisma studio` | 開啟資料庫 GUI 檢視 |
| `npx prisma migrate status` | 檢查 migration 狀態 |

## 重要設定檔

- `next.config.ts` — Next.js 設定（含圖片網域白名單、`allowedDevOrigins`）
- `prisma.config.ts` — Prisma 設定（schema 與 datasource）
- `prisma/schema.prisma` — 資料庫結構定義
- `.env` — 環境變數，見 [[環境變數設定]]

> [!note] 修改設定檔後要重啟
> 改 `.env`、`next.config.ts`、`prisma.config.ts` 後需 **重啟伺服器** 才生效。
> 改一般 `.tsx` / `.ts` 元件則由 Fast Refresh 自動熱更新，免重啟。

## 部署過程踩過的坑

完整紀錄見 [[疑難排解紀錄]]，重點：
1. `node_modules` 安裝不完整 → 用 `npm ci` 重裝
2. `127.0.0.1` 跨來源被擋 → 設定 `allowedDevOrigins`
3. 結帳頁無限迴圈 → cart-provider 用 `useCallback` 穩定函式
