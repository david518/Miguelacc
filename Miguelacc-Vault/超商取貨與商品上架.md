# 📦 超商取貨與商品上架

回到 [[00-總覽 (MOC)]]。相關：[[綠界測試金流]]、[[環境變數設定]]、[[後台管理]]。

2026-06-21 新增的一條龍功能：逐商品設定可否超商取貨、結帳選擇配送方式、綠界物流選店、以及不依賴蝦皮的後台新增商品。

## 1. 商品「可超商取貨」開關

- 資料層：`Product.allowCvsPickup`（Prisma 欄位）
- 後台：商品編輯頁有「可超商取貨（綠界物流）」開關
- 自動判斷（預留）：`deriveCvsEligibility()`（`product-overrides.ts`）
  - 若蝦皮匯入資料含「店到店」字樣 → 自動視為可超商取貨
  - 或重量/尺寸在超商上限內（≤5kg、單邊≤45cm、合計≤105cm）
  - **目前匯入的 `products.json` 沒有這些資訊（重量全 0），所以預設為關，需在後台手動開**

> [!note] 為何用店到店當判斷
> 蝦皮店到店是蝦皮自家物流、不能用於官網訂單；但「蝦皮有開店到店」代表此商品夠小夠輕，
> 可作為「適合用綠界超商出貨」的判斷依據。官網實際出貨走 **綠界物流**。

## 2. 結帳：配送方式

- `ShippingMethodSelector.tsx`：宅配到府 / 超商取貨
- 購物車**所有**商品都 `allowCvsPickup` 時，才可選超商取貨
- 收件地址：宅配必填、超商免填（伺服器端依方式驗證）
- 訂單存：`shippingMethod`、`cvsStoreId/Name/Address`、`cvsLogisticsType`

## 3. 綠界物流：電子地圖選店

- `src/lib/ecpay-logistics.ts`：物流設定、子類型、地圖參數、建立物流訂單
- `GET /api/ecpay/cvs-map`：回傳自動送出的表單，開到綠界電子地圖
- `POST /api/ecpay/cvs-callback`：綠界選店後 POST 回來，用 `postMessage` 傳回結帳分頁並關閉彈窗
- 用**彈出視窗**避免換頁清空購物車（記憶體狀態）
- 超商子類型：`UNIMARTC2C`(7-11) / `FAMIC2C`(全家) / `HILIFEC2C`(萊爾富) / `OKMARTC2C`(OK)

> [!important] 電子地圖的 ServerReplyURL 可用 localhost
> 選店結果是「使用者瀏覽器」POST 回來（非綠界伺服器主動連線），所以本機 localhost 可正常運作。

## 4. 建立物流訂單（出貨單）

- `createCvsShipment()`：付款成功後，於 `POST /api/ecpay/notify` 自動向綠界建立 C2C 超商取貨訂單
- ⚠️ 因為付款的 ReturnURL（server-to-server）連不到 localhost，這步**本機需 ngrok 才能實測**

## 5. 後台新增商品（不依賴蝦皮）

- `/admin/products/new`：直接在 DB 建立商品（`shopeeId = null`）
- 公開頁 `getPublicProducts()` / `getPublicProductById()` 已納入 DB 原生商品
- 商品列表顯示「蝦皮匯入 N ＋ 自建 M」，並有「＋ 新增商品」按鈕
- 編輯：DB 原生商品用 `id` 更新；蝦皮商品維持 override（`isDbNative` 旗標區分）

## ⚠️ 重要：物流用 MD5，金流用 SHA256

> [!danger] CheckMacValue 演算法不同
> - **金流（AIO）**：SHA256（`EncryptType=1`）
> - **物流（Logistics）**：**MD5**（沒有 EncryptType）
>
> URL encode 規則相同（.NET 風格、小寫、空白→`+`），只有雜湊演算法不同。
> 程式用 `ecpayCheckMac(params, key, iv, "md5")` 指定物流用 MD5。一開始物流建單
> 卡在「CheckMacValue驗證錯誤」就是因為誤用 SHA256。
